# 实战游戏路线图

## 选择原则

实战游戏不是越大越好，而是要能训练明确能力。

每个实战项目都要满足：

- 玩法规则清楚。
- 规模可控。
- 有可运行闭环。
- 能自然引入新的 Cocos 概念。
- 完成后能复盘对象关系和数据流。

## 实战 0：Star Catcher

状态：正在进行。

定位：入门训练游戏。

训练目标：

- Node / Component。
- 输入和移动。
- Graphics 绘制。
- Label UI。
- 节点引用。
- 简单距离检测。
- 随机重生。
- 游戏状态机。
- Start / GameOver / Restart 流程。

完成后可以继续加：

- 音效。
- 收集特效。
- 星星 Prefab。
- 连击或时间奖励。
- 最终得分结算。

## 实战 1：Dodge Runner 躲避冲刺

定位：第一个完整反应类小游戏。

玩法：

- 玩家左右移动。
- 障碍物从上往下落。
- 躲避障碍物，存活越久分数越高。
- 碰到障碍物游戏结束。

训练能力：

- Prefab 动态生成。
- 对象池入门。
- 碰撞检测。
- 难度随时间增加。
- 游戏速度和节奏控制。

为什么适合：

这个游戏能自然引出“生成多个对象”和“回收对象”，是 Star Catcher 之后最合适的下一步。

## 实战 2：Brick Breaker 打砖块

定位：碰撞和物理理解训练。

玩法：

- 玩家控制挡板。
- 球反弹击碎砖块。
- 清空砖块通关，球掉到底部失败。

训练能力：

- Collider。
- RigidBody 或自定义反弹。
- 关卡布局。
- 多对象碰撞响应。
- 胜利 / 失败状态。

为什么适合：

它能把“距离检测”升级到更正式的碰撞体系，同时规则仍然简单。

## 实战 3：Top-down Shooter 俯视角射击

定位：动作游戏基础训练。

玩法：

- 玩家用键盘移动。
- 鼠标或方向键控制射击方向。
- 敌人靠近玩家。
- 子弹击中敌人得分。
- 玩家被碰到扣生命或失败。

训练能力：

- 子弹 Prefab。
- 敌人生成。
- 简单敌人 AI。
- 血量系统。
- 冷却时间。
- 对象池。
- 多类对象协作。

为什么适合：

这是从“收集游戏”进入“动作游戏”的关键项目，会显著提高你对游戏对象协作的理解。

## 实战 4：Mini Tower Defense 迷你塔防

定位：系统设计训练。

玩法：

- 敌人沿固定路径前进。
- 玩家放置防御塔。
- 防御塔自动攻击范围内敌人。
- 阻止敌人到达终点。

训练能力：

- 路径系统。
- 目标搜索。
- 攻击频率。
- 升级和金币。
- UI 操作。
- 数据配置。

为什么适合：

它会训练“系统之间如何协作”，比单纯动作游戏更考验架构判断。

## 实战 5：One-screen Platformer 单屏平台跳跃

定位：平台动作训练。

玩法：

- 玩家左右移动和跳跃。
- 收集物品。
- 避开陷阱。
- 到达终点通关。

训练能力：

- 平台碰撞。
- 重力和跳跃。
- 地面检测。
- 动画状态。
- 摄像机或单屏布局。

为什么适合：

平台跳跃是游戏开发中的经典基本功，但比前几个项目更容易踩坑，适合在掌握碰撞后做。

## 推荐顺序

推荐路线：

1. `Star Catcher`：建立 Cocos 基础心智。
2. `Dodge Runner`：学习动态生成和对象回收。
3. `Brick Breaker`：学习碰撞和反弹。
4. `Top-down Shooter`：学习动作对象协作。
5. `Mini Tower Defense`：学习系统设计。
6. `One-screen Platformer`：学习平台动作和动画状态。

## 每个实战项目的交付物

每个实战游戏都应该交付：

- 一份设计文档。
- 一份实现计划。
- 每课复盘文档。
- 可运行 Cocos 场景。
- 清晰 Git 提交历史。
- 技术总结：对象关系、数据流、状态机、踩坑记录。
